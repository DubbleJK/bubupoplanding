이 파일은 대화에서 정리한 **결정·방법·할 일**을 잊지 않도록 저장한 것입니다.

---

## 1. 카카오 채널 자동응답 (키워드)

- **관리 위치:** 카카오톡 채널 관리자 → 자동응답 메시지 / 사용자 발화(최대 14자 등 제한 있음).
- **등록해 둔 사용자 발화 예시 (품목별):**
  - `스티커 문의`
  - `배너 문의`
  - `티셔츠 문의`
- **중요:** 사이트·링크·안내 문구와 **띄어쓰기까지 완전히 동일**해야 자동응답이 맞음.
- 목록에 **「연결되지 않음」**이면 응답 메시지 본문을 채워 저장해 **연결 완료**할 것.
- 저장 후 **본인 폰으로** 채팅에서 위 문구를 그대로 보내 자동응답 테스트.

---

## 2. 당장 안 하기로 한 것 (나중에)

- 추천 패키지 버튼마다 **복사 후 채팅 열기** 같은 웹 쪽 분기는 **보류**.
- **카카오톡 챗봇** 또는 **채팅방 메뉴**로 품목 안내·분기를 정리할 예정.
- 그 구조가 잡히면 랜딩 버튼과 다시 맞추면 됨.

---

## 3. `pf.kakao.com` + `text=` 로 미리 입력 (참고용)

- **형식:**  
  `https://pf.kakao.com/_{채널ID}/chat?text={URL인코딩된문구}`
- 예: 공백은 `%20`, 한글은 `encodeURIComponent()` 사용 권장.
- **장점:** 버튼 하나로 채팅 열고 입력창에 문구가 들어가면, 고객은 전송만 하면 됨. 자동응답 키워드와 문구만 맞추면 연동 가능.
- **주의:**
  - 공식 문서에 항상 명시된 방식은 아님 → 동작 변경 가능성 있음.
  - **모바일 앱에서는 잘 되는 경우가 많고**, PC 웹 등에서는 `text=`가 무시될 수 있음 → **반드시 실제 기기에서 링크 테스트**.
- 추천 패키지 3개에 쓸 때는 위 **스티커 문의 / 배너 문의 / 티셔츠 문의**와 동일한 문자열을 `text=`에 넣을 것.

- 보안: 외부 링크 `rel="noopener noreferrer"`, `next.config.js` 보안 헤더.
